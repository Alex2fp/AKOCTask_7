# Задание: Клиент и сервер на POSIX shared memory

## Цель

Реализовать **две независимые программы** — клиента и сервера — которые обмениваются данными через **разделяемую память POSIX** (`shm_open`, `mmap` и т.д.). Клиент **автоматически** генерирует случайные числа в диапазоне, (0–99), сервер выводит эти числа на экран.

Нужно также реализовать **корректное завершение** работы: при попытке завершить один процесс (клиент *или* сервер) оба корректно завершаются, а сегмент разделяемой памяти удаляется.

---

## Требования

### 1. Общие

- Язык: C.
- Платформа: POSIX (Linux).
- Запрещено использовать SysV shared memory (`shmget`, `shmat` и т.п.), только **POSIX**:  
  - `shm_open`,  
  - `ftruncate`,  
  - `mmap`,  
  - `munmap`,  
  - `shm_unlink`.
- Взаимодействие только через **один объект разделяемой памяти** (имя, например, `"posix-shar-object"` или аналогичное).
- Гарантируется (по условию), что одновременно запущены **ровно один** клиент и **ровно один** сервер. Это можно использовать для упрощения протокола.
