# AKOCTask_7

## Описание программы

`copy32` — утилита на C++ (в стиле C), которая копирует содержимое одного файла в другой, используя только системные вызовы `open`, `read`, `write`, `close`, `fstat` и `fchmod` для работы с файлами. Для буферизации применяется кольцевой буфер фиксированного размера 32 байта, который последовательно используется и при чтении, и при записи. Программа сохраняет права доступа исходного файла. Благодаря этому исполняемые бинарные файлы и скрипты после копирования остаются исполняемыми, а обычные текстовые файлы не получают дополнительных прав.

## Клиент и сервер на POSIX shared memory

В репозитории добавлен пример взаимодействия независимых процессов клиента (`client.c`) и сервера (`server.c`) через POSIX-разделяемую память. Клиент автоматически генерирует случайные числа в диапазоне 0–99 и записывает их в разделяемый сегмент, а сервер считывает и выводит эти значения.

### Сборка

```bash
gcc -std=c11 -Wall -Wextra -pedantic client.c -o client
gcc -std=c11 -Wall -Wextra -pedantic server.c -o server
```

### Запуск

1. В одном терминале запустите сервер: `./server`.
2. В другом терминале запустите клиента: `./client`.

Клиент будет генерировать числа каждые ~200 мс, сервер — выводить их по мере поступления. Корректное завершение обеспечивается обработчиками сигналов `SIGINT`/`SIGTERM`: при остановке любого процесса в разделяемой памяти выставляется флаг `terminate`, и оба приложения завершаются, освобождая сегмент разделяемой памяти.

## Использование

```bash
g++ -std=c++17 -Wall -Wextra -pedantic copy32.cpp -o copy32
./copy32 <исходный_файл> <файл_назначения>
```

## Ожидаемая оценка

Реализованы требования базовой части (до 6 баллов) и опционального задания (+1 балл) — программа корректно работает с текстовыми и бинарными файлами, включая исполняемые, сохраняя их режим доступа. Ожидаемая оценка: **7 баллов**.
